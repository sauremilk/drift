# Decision Templates

Templates for Architecture Decision Records (ADRs) used in this project.

**When to write an ADR:** Changes to signals, scoring logic, output formats, or architecture boundaries.
**Exempt:** Bugfixes, pure refactorings, documentation-only changes.

| Template | Use when |
|---|---|
| [adr-template.md](adr-template.md) | General architecture or process decisions |
| [signal-design-template.md](signal-design-template.md) | Designing a new signal (includes FMEA pre-entry) |

**Lifecycle:** `proposed` → `accepted` | `rejected` | `superseded`. Only the maintainer changes status from `proposed`.
