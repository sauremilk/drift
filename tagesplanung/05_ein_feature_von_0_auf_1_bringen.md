# Ein Feature von 0 auf 1 bringen

## Schnellansicht
- Heute wählen, wenn: Du heute sichtbar etwas Neues liefern willst
- Heute zuerst: Ein kleines Feature mit klarem Mehrwert und engem Scope festlegen
- Feierabendziel: Eine erste nutzbare Version ist implementiert, getestet und erklärbar

## Tagesziel
Nicht nur warten und verbessern, sondern echten Produktfortschritt erzeugen.

## Heute wählen, wenn
- Es fehlt ein klarer Nutzwertschritt für Anwender oder für dich selbst
- Das Projekt ist stabil genug, um neue Funktionalität aufzunehmen
- Du willst nicht nur intern verbessern, sondern sichtbar liefern
- Ein Feature ist klein genug für einen Tag, aber groß genug für echten Mehrwert

## Mögliche Aufgaben für den Tag
- Neue CLI-Option hinzufügen
- Neuen Analysemodus einführen
- Neuen Report oder Export bauen
- Neue Validierungsregel ergänzen
- Neue Integrationsmöglichkeit vorbereiten

## Heute zuerst
- Ein Feature mit klarem Tagesumfang auswählen
- Ergebnis in einem Satz formulieren
- Drei bis fünf Akzeptanzpunkte für heute festlegen

## Tagesablauf
### 1. Startblock am Morgen
- Ein Feature auswählen, das klein genug für einen Tag und groß genug für echten Mehrwert ist
- Das gewünschte Ergebnis in einem Satz formulieren
- Eingaben, Ausgaben und Fehlverhalten grob festlegen

### 2. Analyseblock
- Die Spezifikation in fünf bis zehn Akzeptanzpunkten formulieren
- Agent die betroffenen Stellen und nötigen Änderungen eingrenzen lassen
- Prüfen, was für eine erste nutzbare Version heute realistisch ist

### 3. Umsetzungsblock
- Agent Implementierung, Tests und Doku erstellen lassen
- Die Umsetzung entlang der Akzeptanzkriterien prüfen
- Auf unnötige Scope-Ausweitung verzichten

### 4. Absicherungsblock
- Fehlerverhalten und Randfälle vor dem Abschluss prüfen
- Tests für Erfolgsfall und Fehlerfall ergänzen
- Dokumentation oder CLI-Hilfe auf Nutzbarkeit prüfen

### 5. Tagesabschluss
- Review und Feinschliff selbst durchführen
- Notieren, was heute wirklich geliefert wurde
- Offene Ausbaupunkte für die nächste Iteration sammeln

## Definition von erledigt
- Das Feature ist in einer ersten nutzbaren Form vorhanden
- Akzeptanzkriterien sind weitgehend erfüllt
- Erfolgsfall und Fehlerfall wurden geprüft
- Doku oder Bedienhinweise wurden nicht vergessen

## Guter Agent-Fokus
- Implementiere dieses Feature entlang klarer Akzeptanzkriterien.
- Ergänze Tests für Erfolgsfall, Fehlerfall und relevante Randfälle.
- Aktualisiere Doku oder CLI-Hilfe so, dass das Feature direkt nutzbar ist.
